Launch BAS

Prerequisitesâ€‹

Before running the command you must do the following steps:

  • Ensure you have a dedicated machine that has at least 8 dedicated CPU cores and 32GB RAM, as it needs to run 7 nodes.
  • Make sure you have a wildcard domain *.example.com set to your machine (use a dedicated machine with a public IP).
  • Modify the config.json file to update parameters as you need (you can find all addresses in the keystore folder).

Config file parametersâ€‹

The config file structure is as such:

  • chainId â€” identifier of your BAS chain.
  • validators â€” list of initial validator set (make sure that you have the same list in the docker compose file).
  • systemTreasury â€” address of system treasury that accumulates 1/16 of rewards (might be governance).
    • consensusParams â€” parameters for the consensus and staking.
    • activeValidatorsLength â€” suggested values are (3k+1, where k is honest validators, even better): 7, 13, 19, 25, 31...
    • epochBlockInterval â€” length of an epoch, specified in blocks; recommended length is 1 day (86400/3=28800, where 3s is block time).
    • misdemeanorThreshold â€” after missing this amount of blocks per day validator losses all daily rewards (penalty).
    • felonyThreshold â€” after missing this amount of blocks per day validator goes to jail for N epochs.
    • validatorJailEpochLength â€” how many epochs validator should stay in jail (7 epochs = ~7 days).
    • undelegatePeriod â€” allow claiming funds only after 6 epochs (~7 days).
    • minValidatorStakeAmount â€” how many tokens the validator must stake to create a validator (in ETH).
    • minStakingAmount â€” minimum staking amount for delegators (in ETH).
  • initialStakes â€” initial stakes for the validators (must match with validators list).
  • votingPeriod â€” default voting period for the governance proposals.
  • faucet â€” a map with initial balances for the faucet and other needs.

Launchâ€‹

You have two launching options:

  • Default option
  • Without a balance loader and SSL certificates

Default optionâ€‹

You can check the Makefile to choose more interesting commands, but if you just need to set up everything, just run:

apt update
apt install build-essential socat
git clone https://github.com/Ankr-network/bas-devnet-setup bas --recursive
cd bas
DOMAIN_NAME=dev-02.bas.ankr.com make all

:::hint The variable DOMAIN_NAME should be set to your domain. :::

Deployed services can be access though next endpoints:

  • https://rpc.${DOMAIN_NAME} (port 8545,9546) - Web3 RPC endpoint
  • https://explorer.${DOMAIN_NAME} (port 4000) - Blockchain Explorer
  • https://faucet.${DOMAIN_NAME} (port 3000) - Faucet
  • https://staking.${DOMAIN_NAME} (port 3001) - Staking UI

Without a balance loader and SSL certificatesâ€‹

If you want to run a node without a load balancer and SSL certificates, run:

make all-no-balancer

Exposed portsâ€‹

Docker Compose files expose next ports:

  • 7432 - Blockscout PostgreSQL database
  • 4000 - Blockscout Explorer
  • 3000 - Faucet UI
  • 3001 - Staking UI
  • 8545 - RPC endpoint
  • 8546 - WS endpoint
  • 30303 - Bootnode
